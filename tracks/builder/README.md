# Learn AI: Builder Track

Ship real applications using AI as your development partner. This track is for anyone who wants to build software — whether you're a developer looking to 10x your speed or a non-technical maker turning ideas into working products with AI.

## Who This Is For

**Both technical and non-technical builders welcome:**

- **If you code:** You know at least one language (Python, JavaScript, etc.). You've shipped projects before. You want AI coding tools to handle the boring parts so you focus on architecture, UX, and the hard problems. You'll learn to use Claude Code, Copilot, and agent workflows to ship in days what used to take weeks.

- **If you don't code (yet):** You have ideas and the drive to build. You've maybe done some no-code tools or light scripting. AI is your technical co-founder — it writes the code while you make the product decisions. You'll be surprised what you can ship in 2 weeks.

Both paths converge: everyone ships a working application by the end.

## Program Structure

**5 sessions over 2.5 weeks** (2 sessions/week)

| Session | Topic | Format |
|---------|-------|--------|
| 0 | Pre-work: Setup and Baseline | Async |
| 1 | Foundations: Building With AI | Live 90 min + Async |
| 2 | Scaffold: From Idea to Working Prototype | Live 90 min + Async |
| 3 | Full Stack: Frontend, Backend, Database | Live 90 min + Async |
| 4 | Agents + Ship: Polish, Test, Deploy | Live 90 min + Async |
| 5 | Ship It: Demo Day | Live 90 min |

## Tools You'll Use

| Tool | Purpose | Required? |
|------|---------|-----------|
| Claude / Claude Code | AI assistant, code generation, debugging | Yes |
| GitHub | Version control, deployment | Yes |
| VS Code or Cursor | Code editor | Yes |
| GitHub Copilot | Inline code suggestions | Recommended |
| Vercel / Railway / Fly.io | Hosting and deployment | One of these (free tier) |
| Any framework | React, Next.js, Flask, FastAPI, etc. | Your choice |

## What You'll Build (Capstone)

A working application that:
- Solves a specific problem (for you, your team, or a community)
- Has a user interface (web app, CLI, or API)
- Uses AI meaningfully (not just for building it — AI is part of the product OR the build process)
- Is deployed and accessible via a URL
- Lives in a public GitHub repo

See [capstone/guidelines.md](capstone/guidelines.md) for full details.

## Quick Start

1. Start with [Session 0: Pre-work](curriculum/session-00-prework/)
2. Work through sessions in order
3. Build your capstone alongside the sessions

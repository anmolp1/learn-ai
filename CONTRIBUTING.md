# Contributing to Learn AI

Thanks for your interest in contributing! This is an open program and we welcome help from everyone.

## Ways to Contribute

### Content
- Fix typos, clarify instructions, improve explanations
- Add examples or demo scripts for existing sessions
- Write or improve role-specific track content
- Add resources (articles, tools, tutorials) to the reading list

### New Tracks
- Propose a new role-specific track (open an issue first to discuss)
- Each track should include: a README, example projects, and capstone ideas
- Follow the structure in existing tracks (see `tracks/data-engineering/` for reference)

### Capstone Showcase
- Completed a capstone? Add it to `capstone/showcase/` via PR
- Include: project name, description, link to repo, what AI tools you used

### Translations
- Translate session content to other languages
- Create a `curriculum/{session}/lang-{code}/` directory for translated content

## Process

1. **Open an issue** describing what you want to do
2. **Fork the repo** and create a branch
3. **Make your changes** — keep PRs focused on one thing
4. **Submit a PR** referencing the issue

## Guidelines

- Write for a professional audience that's new to AI integration (not AI research)
- Keep content practical — theory should serve practice
- Include working examples, not pseudocode
- Test any code examples before submitting
- Use plain language — avoid jargon unless you define it

## Code of Conduct

Be respectful, be helpful, be constructive. We're all learning.

# Session 3: Agents and Workflows — Making AI Do the Heavy Lifting

**Format:** 90 min live + async challenge
**Prerequisites:** Session 2 complete
**Goal:** Move from single prompts to multi-step AI workflows

## Session Plan

### Concept (20 min): What Agents Actually Are
- An agent is AI that can take actions, not just generate text
- The loop: Observe → Think → Act → Observe again
- When to use agents vs. simple prompts (complexity threshold)
- Tools and integrations: how agents connect to the real world

### Demo (30 min): Build a Multi-Step Workflow
Build one of these live:

**Option A:** A code review agent that reads a PR, identifies issues, and creates GitHub comments
**Option B:** A content pipeline that researches a topic, drafts an article, and formats for multiple platforms
**Option C:** A data monitoring agent that checks pipeline health and sends alerts

Show the full build process: define the workflow → implement each step → connect them → test end-to-end.

### Tools to Cover
- Claude with tool use / MCP (Model Context Protocol)
- GitHub Actions for AI-powered CI/CD
- Simple agent frameworks (keep it practical, not academic)
- Claude Code / Codex for agentic coding workflows

### Practice (25 min)
- Participants add an agent or automation component to their capstone
- Pair programming encouraged — this is where things get tricky

### Debrief (15 min)
- What worked? What broke? What would you automate next?

## Async Challenge
- Add at least one automated workflow to your capstone project
- Document the workflow: what triggers it, what steps it takes, what it outputs
- Push to GitHub and share in community space

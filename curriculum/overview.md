# Curriculum Overview

## Program Philosophy

This program teaches AI integration through building, not lecturing. Every session follows the same rhythm:

1. **Concept** (20 min) — What's the idea? Why does it matter?
2. **Demo** (30 min) — Watch it done live, messy and real
3. **Practice** (25 min) — Try it yourself with guidance
4. **Debrief** (15 min) — What worked? What didn't? Questions.
5. **Challenge** (async) — Apply it to your own work before next session

## Prerequisites

- Comfortable using a computer for your job (no coding required for some tracks)
- Willing to create free accounts for AI tools (Claude, GitHub, etc.)
- Have a real problem or project in mind from your work

## Tools Used

The program is tool-agnostic where possible, but references these tools as primary examples:

| Tool | Used For | Required? |
|------|----------|-----------|
| Claude (Anthropic) | AI assistant, coding, analysis | Yes (free tier works) |
| GitHub | Version control, project hosting | Yes (free) |
| VS Code / Cursor | Code editor with AI integration | Recommended |
| GitHub Copilot / Codex | AI pair programming | Optional |
| Terraform | Infrastructure-as-code (data eng track) | Track-specific |
| Discord | Community communication | Recommended |

## Session Dependencies

```
Session 0 (Pre-work)
    └── Session 1 (Foundations)
            └── Session 2 (Building)
                    └── Session 3 (Agents)
                            └── Session 4 (Quality)
                                    └── Session 5 (Ship It)
```

Sessions build on each other. Skipping ahead is not recommended for cohort participants, though self-paced learners can jump to specific topics.

## Capstone Project

The capstone project runs alongside the sessions:

- **Session 0:** Think about what you want to build
- **Session 1:** Define the problem and scope
- **Session 2:** Scaffold the first version with AI
- **Session 3:** Add agent/automation components
- **Session 4:** Audit for quality and security
- **Session 5:** Present and celebrate

See [`capstone/guidelines.md`](../capstone/guidelines.md) for full details.

## Adapting This Curriculum

If you're forking this for your own cohort:

- **Timing is flexible.** 5 sessions over 2-3 weeks is the default, but you can stretch to 4-6 weeks for a slower pace.
- **Tracks are modular.** You can run the program with just 1-2 tracks if that fits your audience.
- **Demos should be live.** Recorded demos lose the "this is messy and real" quality that makes the program effective.
- **Cohort size of 10-25 works best.** Smaller allows for more interaction. Larger needs breakout rooms and TAs.

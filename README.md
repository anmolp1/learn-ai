# Learn AI: Integrate AI Into Your Real Workflows

A structured, open-source bootcamp that teaches professionals how to use AI tools and agents to do their actual jobs better — not just play with chatbots.

## What This Is

A cohort-based program where you pick a real problem from your work and solve it with AI. By the end, you have a working project, a GitHub repo to show for it, and the skills to keep building.

This isn't another "intro to AI" course. Each track has its own curriculum, timeline, and capstone — tailored to how AI actually fits into that role.

## Tracks

Each track is a standalone program with its own sessions, tools, and capstone project.

### [Data Engineering Track](tracks/data-engineering/) — *6 sessions / 3 weeks*
Build, test, deploy, and monitor data pipelines using AI as your co-pilot. From writing Terraform configs to debugging dbt models to building AI-powered pipeline monitoring.

**You'll build:** A complete data pipeline — ingestion, transformation, cloud deployment via Terraform, and automated monitoring — all in a public GitHub repo.

**Tools:** Claude, Python, SQL, Terraform, dbt, GCP/BigQuery, GitHub

### [Builder Track](tracks/builder/) — *5 sessions / 2.5 weeks*
Ship real applications using AI as your development partner. For developers who want to build faster and non-technical makers turning ideas into working products.

**You'll build:** A working, deployed web application with a live URL — from idea to production in 2.5 weeks.

**Tools:** Claude / Claude Code, GitHub, Cursor/VS Code, Vercel/Railway, any framework

## How It Works

| Component | Details |
|-----------|---------|
| **Live sessions** | 2 per week, 90 min each. Instructor-led demos + practice. |
| **Async challenges** | Hands-on work between sessions. Apply concepts to your capstone. |
| **Community** | Discord for peer support, sharing progress, asking questions. |
| **Office hours** | Weekly drop-in sessions for when you're stuck. |
| **Capstone project** | A real project you build throughout the program and present at the end. |

## Quick Start

### As a Participant
1. **Pick your track** — [Data Engineering](tracks/data-engineering/) or [Builder](tracks/builder/)
2. **Create your project repo** — Use the [learn-ai-starter](https://github.com/YOUR_USERNAME/learn-ai-starter) template
3. Start with Session 0 (pre-work) in your track
4. Work through sessions in order, building your capstone as you go
5. Present at the final session

### As a Contributor
1. Read [CONTRIBUTING.md](CONTRIBUTING.md)
2. Open an issue or PR
3. Add examples, improve content, or propose new tracks

## Philosophy

1. **Project-first.** Every session ends with something you built.
2. **Track-specific depth.** Each role gets its own curriculum, not a generic one-size-fits-all.
3. **Tools are means, not ends.** We teach patterns. Tools change; patterns endure.
4. **Ship something real.** Your capstone solves a real problem and lives in a public repo.

# Capstone Project Guidelines

## Overview

The capstone project is the core deliverable of this program. It's not a homework assignment — it's a real project that solves a real problem in your work, built with meaningful AI integration.

## Requirements

1. **Solves a real problem** in your domain (not a toy example or tutorial project)
2. **Uses AI meaningfully** — AI should be integral to the workflow, not bolted on
3. **Has a public GitHub repo** with a clear README
4. **Can be demoed in 5 minutes** — keep the scope tight

## Timeline

| Session | Capstone Milestone |
|---------|-------------------|
| 0 (Pre-work) | Think about what you want to build |
| 1 (Foundations) | Define the problem and scope |
| 2 (Building) | Scaffold first version with AI |
| 3 (Agents) | Add automation / agent components |
| 4 (Quality) | Audit, test, and polish |
| 5 (Ship It) | Present to the cohort |

## Scoping Tips

**Too small:** "Use Claude to write a README" — this is a single prompt, not a project.

**Just right:** "Build a pipeline that uses AI to classify incoming support tickets, route them to the right team, and draft initial responses" — clear scope, real problem, meaningful AI use.

**Too big:** "Build a full SaaS product with AI-powered everything" — you have 2-3 weeks. Scope down.

**Rule of thumb:** If you can't explain the project in 2 sentences, it's too big. If you can explain it in 2 words, it's too small.

## Example Projects

| Role | Project | AI Integration |
|------|---------|---------------|
| Data Engineer | Automated data quality monitoring pipeline | AI classifies anomalies, generates alerts, suggests fixes |
| Security | Vulnerability assessment report generator | AI scans configs, categorizes risks, generates remediation steps |
| Builder | Personal finance tracker web app | AI scaffolds the app, writes tests, powers natural-language queries |
| Marketer | Multi-platform content engine | AI generates drafts, adapts tone per platform, creates scheduling plan |
| PM / Ops | Weekly team status report generator | AI summarizes tickets/commits, identifies blockers, drafts report |
| Designer | Design system documentation tool | AI generates component docs from design files, suggests accessibility fixes |

## README Template

Your capstone repo should include a README with at minimum:

```markdown
# [Project Name]

## Problem
What problem does this solve? Why does it matter?

## Solution
What did you build? How does it work?

## AI Integration
What AI tools did you use? How are they integrated?
List specific tools and what role they play.

## Setup
How to run this project locally.

## Demo
Screenshots, GIFs, or a link to a video walkthrough.

## What I Learned
Key takeaways from building this with AI.

## Built With
- [Tool 1]
- [Tool 2]
- ...

## Author
Your name and links.
```

## Evaluation Rubric (Optional)

If your cohort uses grading:

| Criteria | Weight | What "Great" Looks Like |
|----------|--------|------------------------|
| Problem definition | 20% | Real, well-scoped, clearly articulated problem |
| AI integration | 30% | AI is used thoughtfully and adds clear value |
| Execution quality | 30% | The project works, code is clean, output is useful |
| Documentation | 20% | Someone else could understand and reproduce this |
